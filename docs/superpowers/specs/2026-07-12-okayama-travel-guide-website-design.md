# 岡山親子行旅遊指南網站 — 設計文件

日期：2026-07-12
來源內容：`岡山行程.md`（7/19～7/26 岡山／倉敷／小豆島／豐島 親子行）

## 目的與使用情境

- 主要使用情境：**旅途中用手機查詢**當天行程、交通、餐廳資訊，取代目前的 Markdown 文件。
- 小豆島、豐島等地網路可能不穩定，需支援**離線查看**。
- 內容由 `岡山行程.md` 衍生，未來若行程調整（改餐廳、改船班），以修改 Markdown 為準，網站內容手動同步更新。
- 純顯示用途，不需要打勾記錄、留言討論等互動/後台功能。

## 架構總覽

- 純前端靜態網站：Vanilla HTML/CSS/JS，不使用框架、不需要 build 工具。
- 單一頁面應用程式（SPA），依 URL hash 切換畫面（例如 `#/day/2026-07-20`），支援瀏覽器上一頁/重新整理。
- 內容集中於單一資料檔 `data.js`，由 `岡山行程.md` 手動轉換為結構化 JS 物件；畫面依此資料動態渲染，不含任何後端或資料庫。
- 部署到 GitHub Pages（免費、可分享連結、支援 PWA 安裝）。

## 畫面結構

### 首頁選單（漢堡選單）

透過左上角選單開啟，收納**跨天、不屬於單一天**的內容：

- 行程總表（7/19～7/26 八天一覽表）
- TODO 清單（機場接送、貴賓室、豐島美術館門票、esim、保險、餐廳等）
- 伴手禮建議（岡山代表伴手禮、水果甜點、送禮／自用分類）

### 日期視圖（進站預設畫面）

進站時依裝置日期自動跳到當天對應日期；若裝置日期不在 7/19～7/26 範圍內，預設顯示 Day0（7/19）。

- 頂部日期切換器：`‹ 7/20（一）›`，左右箭頭切換前後一天
- 底部四個分頁：
  - **今日**：當天完整時間軸行程（時間、活動、簡短提示）
  - **交通**：當天交通細節（JR/巴士/船班、票價、耗時），含附錄性質的船班時刻表（例：小豆島⇄豐島渡輪時刻）
  - **餐廳**：當天餐廳資訊與優先順序（例：豐島美術館カフェ／いちご家的優先排序與備案）
  - **總覽**：當天內容的濃縮摘要（今日＋交通＋餐廳整合成一頁，方便快速掃過）

多方案行程（例：7/23 豐島 Plan A／Plan B）以 `alternatives[]` 並列顯示於「今日」分頁，不做投票或選擇記錄功能，僅呈現兩案內容與作者建議（「我最後會幫你定成這版」）。

## 視覺風格

**方案 B：溫馨手帳／桃太郎感**

- 底色：米杏色（`#fdf6ec`）
- 點綴色：岡山桃子暖棕橘色調（`#c9915d` / `#e8c4a2` / `#5c3a21`）
- 字體：襯線字體（標題與內文皆可用襯線，營造手寫筆記感）
- 時間軸項目以虛線左邊框分隔，呼應手帳風格
- 底部分頁選中狀態用淺橘底色標示目前分頁

## 離線支援（PWA）

- `manifest.json` 定義 App 名稱、圖示（暫定桃子 🍑 意象呼應岡山名產與桃太郎主題）、啟動畫面、加入主畫面行為。
- Service Worker 採 cache-first 策略，快取所有靜態資源（HTML/CSS/JS）與 `data.js`。
- 快取版本號跟著內容更新遞增，確保重新上線時能抓到最新資料，不會永久卡在舊快取。

## 內容維護流程

- `岡山行程.md` 為行程規劃時的主要編輯文件。
- 內容異動後，由協作者（Claude）手動同步更新 `data.js` 對應欄位。
- 目前資料量（8 天）不大，暫不建立自動化 Markdown → JSON 轉換器；若未來內容持續變動頻繁，可再評估是否值得投入自動化。

## 測試／驗證方式

- 靜態網站無自動化測試框架必要，改採手動核對：
  - 逐天比對 `data.js` 內容與 `岡山行程.md` 是否一致（無漏項、無誤植時間）
  - 手動切換日期、切換四個分頁，確認渲染正確
  - Chrome DevTools 開啟「離線」模式，確認已快取頁面可正常瀏覽
  - 實機（iOS／Android）測試加入主畫面、離線開啟

## 明確排除範圍（YAGNI）

- 不做使用者帳號、後端、資料庫
- 不做打勾記錄、留言討論、方案投票等互動功能
- 不做自動化 Markdown 轉換器（現階段手動同步即可）
- 不支援多語言（僅繁體中文）
